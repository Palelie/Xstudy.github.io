    # 面向对象（基础）

    # 1、方法重载

    条件：

    1. 在同一个类中
    2. 方法名相同
    3. 形参列表不同（只和形参有关：类型、个数、顺序）

    注意，方法重载和返回值类型无关（如果没有进行返回值接收，那就没区别了）；和有无修饰符（static）也无关

    # 2、一些术语

    1. OOA：面向对象分析（Object-Oriented Analy）

    2. OOD：面向对象设计（Object-Oriented Design）

    3. OOP：面向对象编程（Object-Oriented Programming）

    4. 面向对象三大特征：封装、继承、多态

    5. 创建对象实例后将对象地址赋予给一个变量，该变量不叫“对象”，而是叫“引用”，即

      • 对象：堆里面new出来的
      • 引用：是存储对象内存地址的一个变量
      • 总结：只要是存储地址的变量都叫引用

    6. 在类体里变量叫成员变量（存在堆中），在方法（函数）里的变量叫局部变量（存在栈中）

    # 3、内存图

    注意

    在java中，没有引用传递，只有值传递，所有等号都是将右边变量储存的值复制一份赋给左边变量

    # 4、构造方法

    形式：public 构造方法名(形参){方法体}

    注意：

    • 若类中无任何构造方法，在构造对象时会默认提供一个无参数的构造方法（缺省构造器），并给成员变量赋默认值：int为0，string为null，boolean为false
    • 当一个类中手动提供了构造方法，系统将不再提供默认构造方法（但可以自己写一个形参为空的默认构造方法）
    • 若手动写的构造方法的方法体内容为空，系统依然会自动赋默认值
    • 在构造方法中没有被赋值的成员变量会自动赋默认值
    • 构造方法支持重载
    • 构造方法名必须和类名一致
    • 构造方法用于创建对象 ，给实例对象初始化

    使用方式：new 构造方法名(形参)

    # 5、static

    # 方法：

    没有static的方法称为“实例方法”或“对象方法”

    public (static) void ***(){}

    调用方式：

    • 带static的：类名.方法名
    • 不带static的：先new对象，再引用.方法名

    # 变量：

    成员变量也可分为实例变量和静态变量

    • 带static的是静态变量，都是类相关的，访问时采用类名.变量名，不需要new对象
    • 不带static的是成员变量，都是对象相关的，访问时采用对象名.变量名，需要new对象

    注意：

    • 对象方法和实例变量都有可能出现空指针异常
    • 静态变量在类加载时就已经在方法区初始化，不需要new对象就已经开辟出空间
    • 实例变量只有在new对象时才初始化并在堆内存中开辟空间
    • 静态变量也可以用”对象名.变量名“的方式使用，且永远不会出现空指针异常（对象名在执行过程中还是会被替换成类名）
    • 各种变量在JVM中存储的地方
      • 局部变量——栈
      • 实例变量（对象级别）——堆
      • 静态变量（类级别）——方法区

    # 8、this

    1. this是一个变量，也是一个引用

    2. 一个对象一个this

    3. this保存当前对象的内存地址，指向自身，严格意义上讲this代表的是"当前对象"

    4. this储存在堆内存中对象的内部

    5. this只能使用在实例方法中，不能在静态方法中使用（在静态方法中调用实例变量或实例方法需要在该静态方法中先new一个对象再调用，但此时依然不能使用this）

    6. 一般可省略，但在以下情况不可省

    public int getHour(){
        return hour;	//此处this.可以省

    //此处this.不可省略
 	public void setHour(int hour){
        this.hour=hour;		//this.hour是成员变量；hour是局部变量

    # 9、this(实际参数列表)

    作用：该方法可以通过当前构造方法去调用另一个本类的构造方法，以达成代码复用的效果

    public class Date{
	private int hour;
	private int minute;
	private int second;
    
    //添加默认构造函数并给成员变量赋默认值
    public Date(){
        this(1,1,1);		//调用了本类中另一个构造函数
        //相当于以下代码
        //hour=1;
        //minute=1;
        //second=1;
    }
    
    public Date(int hour,int minute,int second){
        this.hour=hour; 
        this.minute=minute;
        this.second=second;
    }
}

    # 11、Object中的toString()方法

    使用方法："引用.toString()" （注意该方法为实例方法，必须先new对象再引用）

    返回类型：string字符串类型

    返回内容：类名@对象地址

    扩展：当使用System.out.println("引用")时，该引用会默认调用toString()方法，打印出"类名@对象地址"

    # 12、方法覆盖/方法重写（override/overwrit）

    # 使用情况：

    子类继承父类后，当继承过来的方法无法满足子类的业务需求时，子类有权利对这个方法进行重新编写,有必要进行"方法的覆盖"

    注意：不要和方法重载（overload）混淆

    # 使用条件：

    1. 两个类必须要有继承关系
    2. 子类中重写的方法和父类的方法除了方法体外需要完全一致才能构成覆盖
    3. 方法的访问权限不能变得更低，但可以变得更高
    4. 重写后的方法不能比之前的方法抛出更多的异常，可以更少

    # 注意事项：

    1. 方法覆盖只针对于方法，与属性无关
    2. 私有方法无法被覆盖
    3. 构造方法无法被继承，所以构造方法也无法被覆盖
    4. 方法覆盖只针对于实例方法，静态方法覆盖没有意义

    # 对Object中的toString()方法进行覆盖：

    由于输出对象的"引用"时，会自动调取"引用.toString()"方法。为了实现直接输出对象时能显示自己想要的信息，就需要对在Object中继承的toString()方法进行重写/覆盖。

    # 方法覆盖中返回值不同的情况(知道有这个语法就行，实际用不上）

    若返回值为基本数据类型，方法覆盖的返回值类型必须相同

    若返回值类型为对象，则被覆盖的方法的返回值类型范围必须比较大（父类）

    class Test{
}

class Animal{
    public Animal get(){	//该方法返回值为Animal对象
        return null;
    }
}

class Cat extends Animal{
    //返回值为Cat的get()方法覆盖
    public Cat get(){		//可以，返回值对象范围变小；若两个方法返回值反过来就不行
        return null;
    }
}

    # 13、多态

    # 引用类型转换：

    1、向上转型：子 ——> 父 （类似于基本数据类型中的自动类型转换）

    ​ 例如：

    ​ Animal a = new Cat(); //其中Animal是父类，Cat是继承了Animal的子类

    2、向下转型：父 ——> 子 （类似于基本数据类型中的强制类型转换,需要加强制类型转换符）

    ​ 例如：

    ​ Cat c = new Animal(); //其中Animal是父类，Cat是继承了Animal的子类

    # 注意：

    1. 引用类型转换只有向上转型和向下转型两种，自动类型转换和强制类型转换是使用在基本数据类型中
    2. 无论是向上转型还是向下转型，两种类型之间必须要有继承关系，（转成接口类型除外）

    # 多态的概念：

    多态表示多种形态/状态：在编译的时候一种形态，在运行的时候另一种形态

    例如父类型引用指向子类型对象，包括了编译阶段和运行阶段。

    编译阶段：绑定父类的方法

    运行阶段：动态绑定子类型对象的方法

    # 举例分析：

    class Animal{
    public void move(){
        System.out.println("动物在跑");
    }
}

class Bird extends Animal{
    public void move(){				//对继承的父类进行方法重写
        System.out.println("鸟在飞");	
    }
    
    public void fly(){				//该方法是子类独有的方法
        System.out.println("飞");	
    }
}

class test{
    Animal a =new Bird();		//表面上a是一个Animal，实际上a是一只鸟
    a.move();   //虽然a的类型为父类Animal，但输出为子类方法的"鸟在飞"
    a.fly();
}

    # 分析：a.move()：

    java程序分为编译阶段和运行阶段

    ​ 编译阶段：对于编译器来说，编译器只知道a的类型是Animal，所以编译器在检查语法的时候，会去Animal.class字节码文件中找move()方法，找到了，绑定上move()方法，编译通过，静态绑定成功。（编译阶段属于静态绑定）

    ​ 运行阶段：运行阶段时，实际上在堆内存中创建的java对象是Bird对象，所以move的时候，真正参与move的对象是Bird，所以运行阶段会动态执行Bird对象的move()方法。这个过程属于运行阶段绑定。（运行阶段绑定属于动态绑定）

    # 分析：a.fly() / 什么时候需要使用"向下转型"？

    结果编译报错：由于fly()方法是子类Bird独有的方法，而a的类型是父类Animal，在编译阶段编译器在Animal.class 字节码中找不到fly()方法，所以静态绑定失败，编译报错,无法运行。（语法错误）

    此时则需要使用"向下转型"（强制类型转换）

    # 向下转型

    依然使用上面代码的例子：

    class test{
    Animal a = new Bird();	//表面上a是一个Animal，实际上a是一只鸟
    a.fly();	//编译报错
    Bird b = (Bird)a;
    b.fly();	//正确
}

    # 分析 ：Cat c = (Cat)a

    1. 编译阶段：编译器检测到a这个引用是Animal类型，而Animal和Cat之间存在继承关系，所以可以向下转型，编译正常
    2. 运行阶段：堆内存实际创建的对象是Bird对象。在实际运行过程中，拿着Bird对象转换成Cat对象就不行了，因为Bird和Cat之间没有继承关系

    # 避免出现类型转换异常的方法：instanceof

    作用：可以在运行阶段动态判断引用指向的对象类型

    语法： "引用" instanceof "类型"

    用法：（instanceof的返回值类型只有true和flase）若c instanceof Cat返回为true，表明c引用指向的堆内存中的java对象是Cat。

    因此以后只要使用到向下转型，都需要先用instanceof进行类型判断

    # 14、super

    # (1) super()

    # super与this相同的地方：

    1. 可以在实例方法和构造方法中出现
    2. 语法：super.或者super()
    3. 不能使用在静态方法中
    4. 大部分情况下都可以省略
    5. super()只能出现在构造方法第一行

    # super关键字的作用：

    通过"子类"的构造方法去调用"父类"的构造方法，初始化父类的类型特征。

    即：要先有父亲才有孩子

    class A{
    public A(){
        System.out.println("A的构造方法实现");
    }
}

class B extends A{
    //B的无参构造方法
    public B(){
    this(123);		//此时该构造方法不会再调用父类构造函数，而是会去指定的构造函数调用super()
    System.out.println("B的无参构造方法实现");
    }
    
    ////B的有参数构造方法
    public B(int n){
    //super();		//子类的构造方法中默认第一句会有super()方法，因此会先去调用父类的构造方法，再回来继续执行子类的构造方法
    System.out.println("B的有参构造方法实现");
    }
}

//当new B()的时候，输出顺序为：
//A的构造方法实现
//B的有参构造方法实现
//B的无参构造方法实现

    # 注意：

    • 既没有this()又没有super()的情况下，子类的构造方法中默认第一句会有 super() 方法
    • this()和super()不能同时出现
    • 由于子类的super()会调用到，所以要保证父类的无参构造方法存在（父类若自定义了构造方法，原来默认的无参构造方法失效）
    • 如果子类每个方法的super(参数)都给了参数，父类可以没有无参构造方法，但还是建议每个类都手动写出无参构造方法
    • 所有方法都会有super()，因为都默认继承了Object类
    • 无论new什么对象，Object类的无参构造方法一定会执行，且一定是最先执行的

    # 内存理解：

    1. 虽然在创建子类对象时，调用了一系列的构造方法，但实际上只创建了一个内存对象
    2. super()的作用：初始化当前对象的父类型特征，并不是创建新对象
    3. super在内存中指向保存的父类型特征（this指向自己）
    4. 理解：虽然我继承了我父亲的肤色，但皮肤只长在我自己身上
    5. super()指向的一系列父类型特征，这些特征彼此之间是包含关系，保存在本对象堆内存空间中

    # (2) super.

    super.的作用类似于this. ，super.在堆内存中指向父类特征（如上图）

    super.不能省略的情况：

    如果父类和子类有相同的实例变量或实例方法，若要指向父类特征，则super.不可省略（此时this.可以省略，默认指向自身）

    若该属性只有父类有，则super.可以省略

    # 注意：

    1. super不可单独使用（this可以），只能super.和super()

      System.out.println(this);		//可以（默认调用this.toString()，返回内存地址）
System.out.println(super);		//不行（报错：super后面要加.）
      1
      2

    2. super不是引用，不保存内存地址，也不指向任何对象。super只是代表当前对象内部那一块父类型特征

      System.out.println(this);		//可以
System.out.println(super.);		//也不行
      1
      2

    3. super.无法调用父类中的私有属性/方法（只能通过公共方法间接调用），私有类型只能在本类中调用
