    # 数组

    # 一、基本特征：

    1. 数组不属于基本数据类型，是一种引用数据类型，其父类也是Object

    2. 数组对象存储在堆内存中，变量名（局部变量）依然存储在栈中

    3. 数组这种数据结构的优点：查找某个下标上的元素效率极高，可以说是查询效率最高的一种数据结构

      原因：(面试题)

      1. 每个元素内存地址连续
      2. 每个元素类型相同，占用空间相同
      3. 将数组首位元素的地址作为数组地址（变量存储的地址），由下标和单元占用大小可以直接计算出元素地址

    4. 缺点：

      1. 随机增删效率低，需要将后面的元素移位
      2. 不能存储大数据量（很难在内存空间上找到一块很大的连续空间）

    # 四、数组扩容

    java中数组长度无法改变，需要扩容的话需要新建一个更大的数组，再把原来的数组拷贝过去，效率很低

    # 五、数组拷贝System.arraycopy()

    通过调用JDK System类中的arraycopy方法，可以进行数组拷贝

    方法中形参中的五个参数：

    System.arraycopy(源数组，源数组起始位置，目标数组，目标数组起始位置，拷贝元素长度)

    //将源数组中的元素选择性拷贝到目标数组中
int[] src = {2,3,7};	//源数组
int[] dest = new int[6];		//目标数组

System.arraycopy(src , 1 , dest , 3 , 2);
for(int i = 0 ; i < dest.length ; i++){
    System.out.println(dest[i]);
}		// 输出结果：0 0 0 3 7 0

    # 六、数组工具类

    java.util下有个Arrays静态工具类

    Arrays.sort(type[],(int from,int to))：对数组索引为from到to范围的元素进行排序（from和to可不写）

    Arrays.binarySearch(type[] a,(int from,int to), type key):使用二分法查询 key 元素值在 a 数组中 from 到 to 索引的元素。出现的索引，如果 a 数组不包含 key 元素值，则返回负数。调用该方法时要求数组中元素己经按升序排列，这样才能得到正确结果。(from和to可不写)
