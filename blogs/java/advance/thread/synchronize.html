    # synchronized同步机制（排它锁）

    # 一、线程同步机制的语法是：

        synchronized(){
        // 线程同步代码块。
    }
    1
    2
    3

    ​ 注意： synchronized后面小括号中传的这个“数据”是相当关键的。这个数据必须是多线程共享的数据。才能达到多线程排队。

    ​ 那么()中的参数写什么？ ​ 那要看你想让哪些线程同步。 ​ 假设t1、t2、t3、t4、t5，有5个线程， ​ 你只希望t1 t2 t3排队，t4 t5不需要排队。怎么办？ ​ 你一定要在()中写一个t1 t2 t3共享的对象。而这个对象对于t4 t5来说不是共享的。

    # 二、示例：使用线程同步机制，解决线程安全问题

    银行账户项目：

    账户类：Account

    public class Account {
    // 账号
    private String actno;
    // 余额
    private double balance; //实例变量。

    //对象（仅作为举例示范）
    Object obj = new Object(); // 实例变量。（Account对象是多线程共享的，Account对象中的实例变量obj也是共享的。）

    public Account() {
    }

    public Account(String actno, double balance) {
        this.actno = actno;
        this.balance = balance;
    }

    public String getActno() {
        return actno;
    }

    public void setActno(String actno) {
        this.actno = actno;
    }

    public double getBalance() {
        return balance;
    }

    public void setBalance(double balance) {
        this.balance = balance;
    }

    //取款的方法
    public void withdraw(double money){

        // 以下这几行代码必须是线程排队的，不能并发。
        // 一个线程把这里的代码全部执行结束之后，另一个线程才能进来。
       
        //方法一：
        //以下举例说明synchronize括号中的参数取值情况
        //Object obj2 = new Object();   局部变量不可以
        //synchronized (obj2) { // 这样编写就不安全了。因为obj2不是共享对象。
        //synchronized (obj) {	//可以，属于该对象的实例变量
        //synchronized ("abc") { // "abc"在字符串常量池当中。
        //synchronized (null) { // 报错：空指针。
        synchronized (this){  //不一定是this，这里只要是多线程共享的那个对象就行。
            double before = this.getBalance();	//获取当前余额
            double after = before - money;		//计算剩下余额
            try {   //模拟网络延迟
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            this.setBalance(after);			//将余额信息返回更新
        }
    }
}

    账户线程类：AccountThread

    public class AccountThread extends Thread {

    // 两个线程必须共享同一个账户对象。
    private Account act;

    // 通过构造方法传递过来账户对象
    public AccountThread(Account act) {
        this.act = act;
    }

    // 多线程并发执行这个方法。
    public void run(){
        // run方法的执行表示取款操作。
        // 假设取款5000
        double money = 5000;
        
        // 取款
        
        //方法二：
        //synchronized (this) { //错误！这里的this是AccountThread对象，这个对象不共享！
        synchronized (act) { // 这种方式也可以，只不过扩大了同步的范围，效率更低了。
            act.withdraw(money);
        }

        System.out.println(Thread.currentThread().getName() + "对"+act.getActno()+"取款"+money+"成功，余额" + act.getBalance());
    }
}

    # 三、线程同步机制原理

    补充：java锁的机制

    在java语言中，任何一个对象都有“一把锁”，其实这把锁就是标记。（只是把它叫做锁。）100个对象，100把锁。1个对象1把锁。

    ​ 银行账户线程同步代码执行原理： ​ 1、假设t1和t2线程并发，开始执行synchronized(){ }代码的时候，肯定有一个先一个后。 ​ 2、假设t1先执行了，遇到了synchronized(){ }，这个时候自动找参数中“共享对象”的对象锁，找到之后，占有这把锁，然后执行同步代码块中的程序，并且在程序执行过程中一直都是占有这把锁的。直到同步代码块代码结束，这把锁才会释放。 ​ 3、假设t1已经占有这把锁，此时t2也遇到synchronized()关键字，也会去占有（）里面的共享对象的这把锁，结果这把锁被t1占有，t2只能在同步代码块外面等待t1的结束，直到t1把同步代码块执行结束了，t1会归还这把锁，此时t2终于等到这把锁，然后t2占有这把锁之后，进入同步代码块执行程序。

    ​ 这样便实现了线程排队执行。 ​ 这里需要注意的是：这个共享对象一定要选好了。这个共享对象一定是你需要排队执行的这些线程对象所共享的。

    ​ 在线程状态角度上看，当线程处于运行状态时遇到synchronize，会让出时间片去锁池里找对象锁（相当于阻塞状态），拿到锁后便回到就绪状态争夺时间片

    # 四、synchronized作为实例方法的关键字

    银行账户类举例：

    public class Account {
    // 账号
    private String actno;
    // 余额
    private double balance;

    public Account() {
    }

    public Account(String actno, double balance) {
        this.actno = actno;
        this.balance = balance;
    }

    public String getActno() {
        return actno;
    }

    public void setActno(String actno) {
        this.actno = actno;
    }

    public double getBalance() {
        return balance;
    }

    public void setBalance(double balance) {
        this.balance = balance;
    }

    //取款的方法
    //将synchronized作为实例方法的关键字
    public synchronized void withdraw(double money){
        double before = this.getBalance(); // 10000
        double after = before - money;
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        this.setBalance(after);
    }
}

    synchronized出现在实例方法上的优缺点：

    • 缺点：
      1. 一定锁的是this，不能是其他的对象。所以这种方式不灵活。
      2. 该方式表示整个方法体都需要同步，可能会无故扩大同步的范围，导致程序的执行效率降低。所以这种方式不常用。
    • 优点：
      • 代码写的少了，简洁了。 如果共享的对象就是this，并且需要同步的代码块是整个方法体， 建议使用这种方式。

    # 五、总结：synchronized的三种使用方式

    synchronized有三种写法：

    第一种：同步代码块 优点：灵活

    	synchronized(线程共享对象){
			//同步代码块;
		}
    1
    2
    3

    ​

    第二种：在实例方法上使用synchronized 表示共享对象一定是this 并且同步代码块是整个方法体。

    第三种：在静态方法上使用synchronized 表示找类锁，用于保证静态变量的安全。 因为类锁永远只有1把。就算创建了100个对象，那类锁也只有一把。

    对象锁与类锁：

    • ​ 对象锁：1个对象1把锁，100个对象100把锁。
    • ​ 类锁：100个对象，也可能只是1把类锁。

public class DeadLock {
    public static void main(String[] args) {
        Object o1 = new Object();
        Object o2 = new Object();

        // t1和t2两个线程共享o1,o2
        Thread t1 = new MyThread1(o1,o2);
        Thread t2 = new MyThread2(o1,o2);

        t1.start();
        t2.start();
    }
}

class MyThread1 extends Thread{
    Object o1;
    Object o2;
    public MyThread1(Object o1,Object o2){
        this.o1 = o1;
        this.o2 = o2;
    }
    public void run(){
        synchronized (o1){
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            synchronized (o2){

            }
        }
    }
}

    Object o1;
    Object o2;

    }
    public void run(){

            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            synchronized (o1){

            }
        }
    }
}
