    # 异常

    # 一、概念

    • java中的异常都是以类的方式存在,每个异常类都可以创建异常对象

    • JVM在执行到错误的地方时会在底层new异常对象，并将new的异常对象抛出，打印输出信息到控制台

    # 二、异常的继承结构

    注意：以下子类只列举出一小部分样例，需要时请查阅帮助文档

    • Object
      • Throwable——不管是错误还是异常都是可抛出的
        • Error（错误）——只要有错误发生，就一定会终止程序并退出JVM，错误是不能处理的
          • IOError
          • VirtualMachineError（虚拟机错误）
            • StackOverflowError
        • Exception（异常）——可以处理的
          • RuntimeException——该类包括其下所有子类都属于"运行时异常",或"未受检异常"、"非受控异常"
            • NullPointerException
            • ClassCaseException
            • IllegalArgumentException
              • NumberFormate Exception
          • Exception的直接子类们——都属于"编译时异常"，或"受检异常"、"受控异常"

    # 运行时异常与编译时异常的区别

    # 相同点：
    • 二者都是发生在运行阶段，编译阶段异常是不会发生的
    • 由于异常的发生就是new对象，而只有运行阶段才能new对象
    # 不同点：
    • 编译时异常：表示在编写程序时需要预先对这种异常进行处理，否则编译器报错，因此得名
    • 运行时异常：在编写程序阶段，可以选择处理也可以选择不处理
    # 实例开发中的区别：
    • 编译时异常：发生概率较高，需要在编写程序时进行预处理
    • 运行时异常：发生概率较低，无需预处理

    # 三、处理异常的两种方式

    1、在方法声明的位置上，使用throws关键字，将异常抛给上一级（即谁调用我，我就抛给谁，且调用者也是这两种处理方式）

    注意：java中异常发生后如果一直上抛，最终抛给main方法，main方法再抛给调用他的JVM，JVM知道这个异常后就会终止程序

    示例：

    public static void main(String[] args) throws ClassNotFoundException{
    doSome();
}

public static void doSome() throws ClassNotFoundException{
    ...
    //若以上代码出现异常便会直接抛出异常，以下后续代码都不会再执行
    ...
}

    2、使用try...catch语句进行捕捉（这件事发生了谁也不知道，因为被我给捕捉了）

    注意：捕捉相当于把异常拦下了，问题真正解决了，调用者也是不知道的

    示例：

    public static void main(String[] args){
    try{	//尝试执行
        doSome();
        //若以上代码出现异常，会直接跳转到catch语句，以下代码不会执行
        ...
    }catch(ClassNotFoundException e){	//此处的e是一个引用，保存的是那个new出来的内存对象的地址
        e.printStackTrace();	//出现异常后走的分支,用于处理异常
    }
    //由于以上异常已被捕捉，即使出现异常以下代码也可以执行
    ...
}

public static void doSome() throws ClassNotFoundException{
    ...

    # 六、异常对象的常用方法

    # 1、获取异常简单的描述信息：

    String msg = exception.getMessage();
    1

    # 2、打印异常追踪的堆栈信息：(一般在catch中使用这个)

    exception.printStackTrace();
    1

    # 示例：

    public static void main(String[] args){
    //这里只是new了异常对象，但是没有将异常对象抛出，JVM会认为这只是一个普通的java对象，因此程序不会终止
    NullPointerException e = new NullPointerException("这是个空指针异常！！！");
    
    //获取异常简单的描述信息：这个信息实际上就是构造方法上面的String参数
    String msg = e.getMessage();
    System.out.println(msg);	//这是个空指针异常！！！
    
    //打印异常追踪的堆栈信息
    e.printStackTrace();
}

    从上往下看，不用看SUN公司的代码信息（异常是一级级抛出，最先抛出的最先打印）

    栈顶信息即为抛出异常的最底层程序

    # 七、finally子句

    1. finally必须和try一起出现，不能单独编写
    2. try可以只和finally联合而不出现catch
    3. try不能单独使用
    4. 在finally子句中的代码是最后执行的，并且是一定会执行的
    5. 通常在finally语句块中完成资源的释放/关闭
    6. 即使try语句块中的代码出现异常，finally中的代码也会正常执行
    7. 即使try语句中有return语句，也会先执行finally语句，最后再执行return
    8. 但try语句中如果是System.exit(0)，则finally语句不会执行（直接退出JVM）
    try{
    ...
}catch(){
    ...
}finally{
    ...

    # 八、final、finally、finalize

    # final：（关键字）

    • final修饰的类无法继承
    • final修饰的方法不能被覆盖
    • final修饰的变量不能重新赋值

    # finally：（关键字）

    和try一起联用，finally语句块中的代码一定会被执行

    # finalize：（标识符）

    Object类中的一个方法，垃圾回收器GC回收前进行调用

    # 九、自定义异常

    # 第一步：编写一个类继承Exception（异常概率高的继承这个）或者RuntimeException
    # 第二步：提供两个构造方法，一个无参数的，一个带有String参数的
    public class MyException extends Exception{
    public MyException(){
        
    }
    public MyException(String s){
        super(s);
    }

    # 十、实际应用

    在实际开发中，要用抛出异常去取代以前面对发生错误时的输出信息+return的模式

    补充：

    throws：在方法声明处使用，表示上报异常给调用者

    throw：手动抛出异常

    //调用栈
public static void main(String[] args){
     MyStack s = new MyStack();
     String a = "abcd";
    try {			//异常捕捉
        s.push(a);
    }catch (MyStackException e){
        e.getMessage();
    }
    
     try {			//异常捕捉
            s.pop();
        } catch (MyStackException e) {
            e.getMessage();
        }
    
}

//自定义异常
public class MyStackException extends Exception{
    MyStackException(){}

    MyStackException(String s){
        super(s);
    }
}

//模拟栈
public class MyStack {
    int i;

    Object[] obj;

    public Stack(){
        obj = new Object[2];
    }

    public Stack(Object[] obj){
        this.obj = obj;
    }

    /*用异常方法改良前
    public void push(Object ob){
        if(i == obj.length){
            System.out.println("栈空间已满，无法压栈");
            return;
        }
        obj[i++] = ob;
    }

       public void pop(){
        if(i == 0){
            System.out.println("空栈无法弹栈");
            return;
        }
        i--;
    }
     */

    //用异常方法改良后
    public void push(Object ob) throws MyStackException{		//要抛出异常，不要自己抛又自己捕捉
        if(i == obj.length){
           throw new MyStackException("栈空间已满，无法压栈");	//手动抛出异常
        }
        obj[i++] = ob;
    }

    public void pop() throws MyStackException {				//抛出异常
        if(i == 0){
            throw new MyStackException("空栈无法弹栈");		//手动抛出异常
        }
        i--;
    }

    public void show(){
        for(int n = 0 ; n < i ; n++){
            System.out.println(obj[n] + " ");
        }
    }
}
