    # List实现类

    # 一、ArrayList（数组/非线程安全）

    # 1、特点和两种常用构造方法

    • 底层是Object[]数组
    • 默认初始化容量是10（底层先创建一个长度为0的数组，当添加第一个元素的时候，初始化容量10）
    List list1 = new ArrayList();		//构造方法1:数组长度是10
list1.size();	//0

List list2 = new ArrayList(20);		//构造方法2:数组长度是20
list2.size();	//0
    1

    集合的size()方法获取的是集合当前的元素个数，而不是获取集合容量

    # 2、扩容

    每次扩容都是原容量的1.5倍（源码：原容量+原容量>>1)

    # 3、优化

    初始化时给的预估值，减少扩容次数

    # 4、优缺点（同数组）

    优点：检索效率高

    缺点：随机增删效率低（向数组末尾添加元素效率还是很高，不受影响）

    # 5、ArrayList是最常用的集合

    原因：平时一般都多检索，添加元素时也是从末尾添加

    # 6、第三种构造方法：将其他集合转化为List集合

    Collection c = new HashSet();
c.add("abc");
c.add(2);

List myList = new ArrayList(c);		//将HashSet集合转化为List集合

    # 二、LinkedList（双向链表）

    # 1、链表特点

    优点：增删效率高

    缺点：检索效率低（需要从头开始遍历查找）

    # 1、下标问题

    LinkedList底层虽然是链表，但其属于List类（有序，可重复），也是有下标的

    List list = new LinkedList();
list.add("abc");
list.add("xyz");

Object obj = list.get(1);	//xyz

    注意：

    ArrayList之所以检索效率高，并不是因为他有下标，而是由他底层是数组所决定的

    LinkedList虽然也有下标，但底层是链表，每次检索还是需要从头遍历，检索效率低

    # 三、面向接口编程

    不管是LinkedList还是ArrayList，写代码的时候不需要关心具体是哪个集合

    因为要面向接口编程，所以调用的都是List接口中的方法

    List list = new ArrayList();	//这样写表示底层用了数组
List list = new LinkedList();	//这样写表示底层用了双向链表

    # 四、Vector（数组/线程安全）

    # 1、特征

    • 初始容量：10
    • 扩容倍数：原容量的2倍
    • Vector中所有方法都是线程同步的，都带有synchronized关键字，是线程安全的
    • 效率较低，现在线程安全已经有其他解决办法，使用较少

    # 五、Collections

    区分：

    Collection：集合接口

    Collections：集合工具类

    # 将线程不安全的ArrayList集合转化为线程安全的

    List list = new ArrayList();	//非线程安全的

Collections.synchronizedList(list);		//转变为线程安全的
