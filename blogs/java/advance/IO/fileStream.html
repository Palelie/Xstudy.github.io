    # 文件字节IO流（重点掌握）

    # 一、java.io.FileInputStream

    # 1、概述

    • 文件字节输入流，万能的，任何类型的文件都可以采用这个流来读
    • 以字节的方式，完成输入的操作，即完成读的操作（硬盘——>内存）
    • 父类继承了Exception（编译时异常），使用时需要处理异常

    # 2、读取示例read()方法

    public static void main(String[] args){
    FileInputStream fis = null;
    try{
        //创建文件字节输入流对象
        //文件路径：C:\Users\r\Desktop\temp
        
        fis= new FileInputStream("C:\\Users\\r\\Desktop\\temp");
        
        //由于'\'表示转义，IDEA会自动将 '\' 转换为 '\\'
        //也可写成"C:/Users/r/Desktop/temp"
        
        //读取文件
        int readData = 0;
        while((readData = fis.read()) != -1){
            System.out.println(readData);
        }
        
    }catch(FileNotFoundException e){
        //此处捕捉的是FileInputStream()抛出的异常
        e.printStackTrace();
    }catch(IOException e){
        //此处捕捉的是read()抛出的异常
        e.printStackTrace();
    }finally{
        //在finally语句块中确保流一定关闭
        
        if(fis != null){	//避免空指针异常
            
            //关闭流的前提是：流不是空
            try{
                fis.close();
            }catch(IOException e){
                e.printStackTrace();
            }
        }
    }
}

    # read()方法解析：

    1. read()是属于FileInputStream类的方法
    2. 该方法返回值类型是int，数值为数据字节
    3. 需要单独处理异常
    4. 一开始指针指向读取数据的前一位（即为空指针），每调用一次方法，指针便往后移动一位并输出

    # read()方法缺点：

    1. 一次只能读取一个字节byte，这样内存和硬盘交互太频繁，基本上时间/资源都耗费在交互上面了
    2. 指针每次读取一个数据都要从硬盘跑向内存
    3. 因此一般不用

    # 改进：int read(byte[] b)

    • 一次最多读取b.length个字节，减少硬盘和内存的交互，提高程序的执行效率
    • 该方法返回值是每次读取到的字节数量（不是字节本身）
    • 每次读取的数据都会从头覆盖数组，但没覆盖到的不会清空
    //假设文件中的数据为abcdef

fis= new FileInputStream("C:\\Users\\r\\Desktop\\temp");

byte[] bytes = new byte[4];	//准备一个4个长度的byte数组，一次最多读取4个字节

int readCount = fis.read(bytes);
System.out.println(readCount);		//第一次读到4个字节
System.out.println(new String(bytes));	//abcd

readCount = fis.read(bytes);
System.out.println(readCount);		//第二次读到2个字节
System.out.println(new String(bytes));	//efcd

readCount = fis.read(bytes);		//一个字节都没有读到返回-1
System.out.println(readCount);		//-1

    # 最终版本（重点掌握）

    public static void main(String[] args){
    //创建输入流对象
    FileInputStream fis = null;
    
    //读取文件（重点）
    try{
        fis= new FileInputStream("C:\\Users\\r\\Desktop\\temp");
        //准备一个byte数组
        byte[] bytes = new byte[4];
        
        //读取数据到byte数组中并输出
        int readCount = 0;
        while((readCount = fis.read(bytes)) != -1){
            System.out.print(new String(bytes,0,readCount));	//选择性地转换并输出
        }
    }catch(FileNotFoundException e){
        //此处捕捉的是FileInputStream()抛出的异常
        e.printStackTrace();
    }catch(IOException e){
        //此处捕捉的是read()抛出的异常
        e.printStackTrace();
    }finally{
        //在finally语句块中确保流一定关闭
        
        if(fis != null){	//避免空指针异常
            //关闭流的前提是：流不是空
            try{
                fis.close();
            }catch(IOException e){
                e.printStackTrace();
            }
        }
    }
}

    # 3、int available()

    返回流当中剩余的没有读到的字节数量

    //一次读取所有剩下未读取的字节
byte[] bytes = new byte[fis.available()];
int readCount = fis.read(bytes);
System.out.println(new String(bytes));

    1. 这种方法不需要循环，可以一次性读取文件中的所有字节
    2. 不太适合太大的文件，因为byte[]数组不能太大

    # 4、long skip(long n)

    跳过几个字节不读

    //直接改变“指针”指向
fis.skip(4);

    # 二、java.io.FileOutputStream

    # 1、概述

    • 文件字节输出流，负责写入
    • 从内存到硬盘

    # 2、write(byte[] b)

    将byte数组中的所有字节写入此文件输出流中

    实现方法和注意事项同下

    # 3、write(byte[] b , int off , int len)

    将指定byte数组中从偏移量off（起始位置）开始的len个字节写入此输出流

    import java.io.FileOutputStream;
import java.io.IOException;

public class IOtest {
    public static void main(String[] args) {
        FileOutputStream fos = null;
        try{
            //创建输出流对象
            fos = new FileOutputStream("src/myfile");

            //开始写
            byte[] bytes = {97,98,99,100};

            //将byte数组全部写出
            fos.write(bytes);

            //将byte数组的一部分写出
            fos.write(bytes,0,2);

            //写完之后一定要刷新
            fos.flush();
        }catch(FileNotFoundException e){
            e.printStackTrace();
        }catch(IOException e){
            e.printStackTrace();
        }finally{
            if(fos != null){
                try{
                    fos.close();
                }catch(IOException e){
                    e.printStackTrace();
                }
            }
        }
    }
}

    注意：

    每次执行带有IO流的class文件时：

    1. 以fos = new FileOutputStream("src/myfile");方式创建的输出流在写入时会将原文件清空，再重新写入（只有重新执行程序时才会）
    2. 写成fos = new FileOutputStream("src/myfile",true);，则以追加的方式在文件末尾写入
    3. 如果文件不存在则会自动新建文件再写入

    # 三、实际应用——文件复制

    1. 使用FileInputStream + FileOutputStream完成文件的拷贝
    2. 拷贝的过程应该是一边读一边写
    3. 使用以上的字节流拷贝文件的时候，文件类型随意，万能的。什么样的文件都能拷贝
    4. 该方法无法拷贝整个目录
    import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.io.FileOutputStream;
import java.io.IOException;

public class copy {
    public static void main(String[] args){
        FileInputStream fis = null;
        FileOutputStream fos = null;
        try{
            //创建一个输入流文件
            fis = new FileInputStream("01.txt");
            //创建一个输出流文件
            fos = new FileOutputStream("02.txt");

            //核心步骤：一边读，一边写
            byte[] bytes = new byte[1024 * 1024];   //1MB（一次最多拷贝1MB）
            int readCount = 0;
            while((readCount = fis.read(bytes)) != -1){
                fos.write(bytes,0,readCount);
            }

            //输出流最后要刷新
            fos.flush();
        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }finally {
            //注意文件关闭要分开try
            //一起try的时候，其中一个出现异常，可能会影响到另一个流的关闭
            if(fos != null){
                try{
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if(fis != null){
                try{
                    fis.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}
