    # 面向对象（进阶）

    # 一、final

    # 1、作用：（总结一句话，final修饰的不能进行修改）

    • 给类加上final关键字后，该类将无法再被继承
    • 给方法加上final关键字后，该方法无法再被覆盖（重写）
    • 给局部变量加上final关键字后，该变量一旦赋值就不能再赋值了（只能赋值一次）
    • 若变量为引用，当该引用指向一个对象后，永远只能指向该对象，也不能指向null(该对象不会被垃圾回收器回收，直到当前方法结束才会释放空间)。注意，final修饰的引用指向的对象内部数据修改不受影响（数组、String等）
    • 给实例变量加上final后，系统不再赋默认值，要求程序员手动赋值（可以在变量后面直接加等号赋值，也可以通过构造函数赋值）

    2、用法：public final class String{}、public final void add(){}、（private） final int i、final People p

    # 3、注意：

    如果要用final修饰实例变量，绝大部分情况都要和static搭配使用。

    原因：

    1. final修饰的变量永远只能取固定值，每new一个对象都要在堆内存给他分配空间，会造成空间浪费。所以用static修饰，只存放一个在方法区中
    2. static修饰的变量是可以被修改的，用final修饰后才是真正意义上的"常量"

    即static final int i，用static final修饰的变量称为"常量"。

    # 二、抽象类

    # 概念：

    抽象类是将类与类之间的共同特征提取出来形成的类。

    # 语法：

    [修饰符列表] abstract class 类名{
	类体;
}
    1
    2
    3

    # 特征：

    1. 抽象类也属于引用数据类型
    2. 由于类本身是不存在的，所以抽象类无法创建对象（实例化）,是用来被子类继承的
    3. 抽象类的子类可以是抽象类
    4. 抽象类虽然无法实例化，但是抽象类有构造方法，这个方法是供子类使用的

    # 三、抽象方法

    # 概念：

    抽象方法表示没有实现的方法，即没有方法体的方法

    # 语法：

    public abstract void doSome();(可以有形参)

    # 特征：

    1. 没有方法体，以分号结尾
    2. 前面修饰符列表中有abstract关键字
    3. 抽象类中不一定有抽象方法，但抽象方法一定在抽象类中
    4. 一个非抽象的类继承抽象类，必须将抽象类中的抽象方法实现（重写/覆盖）

    # 四、接口

    # 特征：

    1. 接口也是一种"引用数据类型"

    2. 编译之后也生成一个class字节码文件

    3. 接口是完全抽象的（抽象类是半抽象的）,或者也可以说接口是特殊的抽象类

    4. 接口与接口/接口与类之间支持多继承，一个接口可以继承多个接口，弥补了类与类之间只能单继承的缺陷

    5. 接口多继承并实现多态情况下，若要调用其他接口的方法，需要向下转型，即需要强制转换

    6. 接口中只有两样东西：常量、抽象方法（不能有方法体，{}也不行，要分号结尾）

    7. 接口中所有的元素都是公开的（public修饰）

    8. 接口中的抽象方法定义时，public abstract修饰符可以省略

    9. 接口中的常量定义时，public static final修饰符可以省略

    10. 若一个类既要继承类，又要实现接口，应该先写extends再写implements

    11. 类若强制转换成接口，不需要有继承关系也能进行转换（但需要进行实现）

      interface Eat{
    void eat();
}

class Animal{}

class Cat extends Animal implements Eat{	//猫类实现了接口，但没有继承该接口
    public void eat(){};	//此处要设
}

class test{
    public static void main(String[] args){
        Animal animal = new Cat();
        if(animal instanceof Eat){			//可以用instanceof判断是否实现了该接口
            Eat e = (Eat)animal;			//可以直接从父类强制转换为子类实现的接口类型
            e.eat();
        }
    }
}

    # 语法：

    [修饰符列表] interface 接口名{}

    # 注意：

    类和类之间叫做 "继承" ， 类和接口 之间叫做 "实现" （依然可以看作是继承）

    # implements关键字：

    类"实现"（继承）接口不用extends，而是用implements

    # 抽象类与接口的区别（语法上）：

    1. 继承抽象类用extends；实现接口用implements
    2. 抽象类是半抽象的；接口是完全抽象的
    3. 抽象类中有构造方法；接口中没有构造方法
    4. 抽象类只能单继承；接口可以多继承
    5. 抽象类不一定有抽象方法；接口只有抽象方法和常量

    # 五、Object类中常用方法

    # 1. protected Object clone() 负责对象的克隆

    # 2. public native int hashCode() 获取对象哈希值的一个方法

    • 该方法不是抽象方法，带有native关键字，底层直接调用C++程序

    # 3. boolean equals(Object obj) 判断两个对象是否相等

    一般需要重写，源码中是直接用"=="来比较对象，只能比较两个对象的内存地址是否相同

    拓展：

    String类型数据比较最好使用String类型的equals()方法进行比较（无需再手动重写）

    //这种写法可以直接用"=="进行比较
String s1="abc";
String s2="abc";

//这种写法只能用equals()进行比较
String s1=new String("abc");
String s2=new String("abc");

s1==s2;		//只适用于第一种初始化方法的String类型数据
s1.equals(s2);		//通用方法，建议使用

    建议使用的效率较高的equals()重写方式

    class Student{
    int no;
    String id;
    
    public boolean equals(Object obj){		//注意此处必须一模一样，否则不是重写
    	if(obj == null || !(obj instanceof Student) )  return false;
        if(this == obj) return true;		//此处比较内存地址
        Student s = (Student) obj;			//向下转型
        if(this.no == s.no && this.id.equals(s.id))  return true;
	}
}

    # 4. String toString() 将对象转换为字符串形式

    一般该方法需要重写，直接输出对象时会自动调用

    # 5. protected void finalize() 垃圾回收器负责调用的方法

    • 该方法没有方法体，只有一个{}
    • 当一个对象即将被垃圾回收器GC回收时，会自动调用finalize()方法
    • 如果想要在对象被回收前执行一些代码，就需要重写该方法（类似于静态代码块）
    • GC只有当垃圾够多时/需要启动时才会启动，用System.gc()方法也只能提高启动的概率

    # 六、内部类

    # 静态内部类：类似于静态变量

    public class MyClass{
    
    //声明一个静态内部类
    private static class InnerClass{
        //1、静态方法
        public static void m1(){}
        //2、实例方法
        public void m2(){}
    }
    
    public static void main(String[] args){
        //1、调用静态内部类的静态方法
        MyClass.InnerClass.m1();
        
        //2、调用静态内部类的实例方法
        //创建静态内部类对象
        MyClass.InnerClass mi = new MyClass.InnerClass();
        //调用方法
        mi.m2();
    }
}

    # 匿名内部类：

    # 不使用匿名内部类的写法
    class Test{
    public static void main(String[] args){
        MyMath mm = new MyMath(); 
        mm.mySum(new computeImpl(),100,200);	//由于接口不能直接new，所以需要编写接口实现类并new一个（多态）
    }
}

//定义接口
public interface compute{
    int sum(int x,int y);
}

//区别：需要写实现类（实现接口）
class computeImpl implements compute{
    public int sum(int x,int y){
        return x+y;
    }
}

//调用接口
class MyMath{
    public void mySum(compute c,int x,int y){
        int res = c.sum(x,y);
        System.out.println(res);
    }
}

    # 使用匿名内部类的写法
    class Test{
    public static void main(String[] args){
        MyMath mm = new MyMath(); 
        mm.mySum(new compute(){					//由于没有实现接口的类，所以使用匿名内部类
            public int sum(int x,int y){		//即mm.mySum(new compute(){方法实现},100,200);
                return x+y;
            }
        },100,200);
    }
}

//定义接口
public interface compute{
    int sum(int x,int y);
}

//调用接口
class MyMath{
    public void mySum(compute c,int x,int y){
        int res = c.sum(x,y);
        System.out.println(res);
    }
}
