    # 步骤与最小生成树的Prim算法类似——每次都把一个路径最短的结点加入集合

    # 关键——与最小生成树算法prim的区别：

    • Prim算法：每次加入新结点后，更新的是整个已归纳的顶点集合与所有未归纳的顶点之间的最短距离

    • Dijkstra算法：每次加入新的结点后，更新的是初始结点与所有未归纳的顶点之间的最短路径（需重新计算起点到其他尚未纳入集合的点的距离，是否会因为加入新的这个点后变的更近（或者原本没路，加了新的点后有路了））。然后再和prim算法一样，继续从集合中选出与剩余各点相距最短的边，将新的结点纳入集合
